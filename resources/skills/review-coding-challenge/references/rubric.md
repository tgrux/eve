# Take home grading rubric

A simplified, non-exact version of this rubric will be provided to the candidate so they know the categories on which they are being evaluated. Please remember if you are grading something that is outside of your core competency for instance you are reviewing a React code submission, but you mostly work in RoR please take extra time to try and understand the language or framework and ask for help if necessary.

## Code Completion
Note: Completion evaluated vs the expected completion level based on the role they are applying to, as outlined in the code challenge.

Red X
The code does not compile.
The solution does not solve the main problem.
There's a significant lack of efficiency in the code.

Thumbs Down
The code compiles without any errors.
The solution solves the problem.
The efficiency of the code could be improved.

Thumbs Up 
The code compiles without any errors.
The solution solves the problem.
The code is reasonably efficient but might have some areas of improvement.
Fails gracefully and has some error handling, but does not cover most cases.

Star
The code compiles without any errors.
The code is optimized for efficiency.
The code contains most error cases.

## Tests
Red X
No tests or only the generated tests are provided.
The test coverage is minimal or non-existent.

Thumbs Down
Few tests are provided, covering only the most obvious cases.
Test flows could be simplified.

Thumbs Up
Good number of tests covering various scenarios.
Test flows are mostly straightforward.

Star
Comprehensive tests covering all possible edge cases.
Test flows are simple and easy to understand.

## Communication

Red X
No documentation, very minimal information, or is wrong.
Inconsistent communication about process, intent and/or challenges.
Does not describe how to run the system.
No information on environment setup or how to run the code and tests.
The code is not readable

Thumbs Down
Basic documentation provided.
Attempted to communicate the process, intent and challenges with their solution.
Describes how to run the system but lacks clarity.
Some information on environment setup and running code/tests but may be incomplete.
The code is somewhat readable but could be improved.

Thumbs Up
Comprehensive documentation with few areas of improvement.
Communication was clear and concise.
Clearly describes how to run the system.
Details on environment setup and running code/tests are mostly clear.
The code is mostly easy to follow.

Star
Exceptional and thorough documentation.
Clearly communicated their solution while also demonstrating tangential knowledge by explaining the pros/cons of various approaches.
Detailed and clear instructions on how to run the system.
Precise steps on environment setup, running the code, and tests. Includes assumptions and future improvements.
The code is exceptionally readable and easy to follow.

## Code Sophistication
Red X
The code is difficult to understand.
The code is cluttered and messy.
Does not follow language conventions or best practices.

Thumbs Down
Some parts of the code are clean, while others are not.
Follows some language conventions and best practices, but not consistently.

Thumbs Up
The code is generally clean (for example follows SOLID principles) with minor areas of improvement.
Follows language conventions and best practices consistently.
The code has good encapsulation

Star
The code is clean and well-structured.
Follows language conventions and best practices consistently.
The code follows a pattern that fits well for the problem (or explains how the problem would naturally expand to fit the pattern)

# FAQ:

Does this apply to all role levels, Associate Software to Principal Engineer etc? Yes
A junior software engineer can have a “3”, the same as a staff engineer can have a “3”, but the “3” scales to each role. For instance a junior engineer only does until story 2. So the tests, completion will naturally scale, and the type of code pattern might not be as complex as a staff engineer. For communication there could be some difference, for instance, a few more assumptions on the environment might be acceptable, but the usage of language and clarity of communication is still a must.

What is the minimum score for each part of the rubric, to be considered a pass to the next interview stage?
Code Completion - 3
Tests - 3
Communication - 3
Code Sophistication - 3
